# MVP Mobile App Planner Agent

A **single-responsibility planning agent** that helps you refine a **mobile app MVP idea** into a
clear, technical, builder-ready specification.

This agent:

- **Asks focused clarifying questions** about users, use cases, constraints, and success metrics.
- **Generates a structured MVP + technical requirements spec** using modern mobile app practices.
- **Outputs one thing only** at the end: a **complete, self-contained prompt string** that you can
  pass directly to a separate *builder agent* (e.g., another script or external LLM).

The planner is available in two modes:
- **Web UI** (recommended): Interactive chat-based interface built with Streamlit
- **CLI**: Command-line interface for terminal-based workflows

Built with `langchain-openai` and the OpenAI API.

## Requirements

- Python 3.12+
- OpenAI API key (`OPENAI_API_KEY`)
- `uv` package manager (recommended) or `pip`

## Install

From the `mvp-mobile-agent` directory:

```bash
uv sync
```

Or with `pip`:

```bash
pip install -e .
```

## Configuration

Create a `.env` file in this directory with:

```bash
OPENAI_API_KEY=your-api-key-here
OPENAI_MODEL=gpt-4o-mini
```

- `OPENAI_MODEL` is optional; defaults to `gpt-4o-mini` if not set.

Launch the interactive web interface:


## High-Level Flow

1. **Idea intake**
   - You type a high-level description of your app.
2. **Clarification**
   - The planner asks targeted questions (users, use cases, platforms, budget/timeline, tech prefs,
     KPIs, competitors).
   - You answer in one block of text.
3. **Requirements synthesis**
   - The agent turns your idea + answers into a structured MVP and technical spec:
     - Problem, users, value prop
     - Key user journeys
     - MVP features vs non-goals
     - Platforms and stack hints
     - Constraints and success metrics
4. **Builder prompt generation**
   - Using that spec, the planner generates a **single, well-structured prompt** for a builder agent.
   - The prompt bakes in:
     - Modern mobile app stack knowledge (React Native, SwiftUI, Kotlin/Compose, backend options,
       CI/CD, testing)
     - Clear role/instructions for the builder
     - The full spec as context
   - The program prints **only this final prompt** at the end, ready to be copyâ€‘pasted.

## Notes

- This agent is intentionally **single-purpose**: it plans and generates a builder prompt; it does
  **not** generate code.
- You can use the final prompt with:
  - Another Python agent,
  - A notebook,
  - Or directly in your LLM IDE / chat client as the system or user message.
- To extend the system, you would typically:
  - Keep this planner as-is,
  - Add a separate `mvp-builder-agent/` that consumes the plannerâ€™s prompt and focuses purely on
    architecture and implementation planning or codegen.
